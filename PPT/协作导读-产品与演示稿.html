一、背景、痛点与客群（司库现金流语境）

1.1 业务背景：为什么「预测、计划、实付」容易三张皮

集团司库的典型一天里：银行与 ERP 流水在多个账户与法人之间分散；预测与场景常活在 FP&A 的 Excel 或咨询交付的幻灯里；资金计划与执行对账又落在共享中心或邮件批复里。三者若不同源、不同步、不回写，就会出现：会上说的「下周缺口」对不上财务系统里的头寸；投委会上一条曲线，台下没人敢拍板执行；接口一断，看板还在闪绿灯——其实是空数。

1.3 产品一句话与价值边界

资金预测智能体帮企业在可信流水之上做分析与预测，并把结果落到资金计划与对账；多法人、多币种时补外汇敞口视角；主数据与集成保证「同一套数」；两阶段模型把方法论钉在界面上；DataAgent 在抽屉里问数、问因——无 Key 时仍可走工具摘要。Agent 不代替司库决裁；多源冲突与覆盖策略须可审计（见详细版末页四追问）。

典型客户：集团司库、资金部、FP&A、财务总监办公室；有跨境收支时 CFO 会关注汇率敏感叙事。

常见需求（可对照投标/访谈）：

  • 可解释：预测不是黑箱一个数，要有区间、假设、可回看的逻辑。
  • 可执行：预测要能进计划、能对执行偏差，否则「预测只活在 PPT」。
  • 可治理：多数据源时谁优先、人工覆盖是否留痕；万级数据下界面仍可用。
角色关心什么
司库经理头寸、缺口、下周能不能付、银行额度与集中度。
资金分析师模型假设、回测、场景对比、表格下钻。
CFO投委会 / 董事会上「钱够不够、风险在哪」三句话 + 一张总览。
IT / 数据接口、同步、主数据映射、审计字段。

5.1 总览看板

Live · 总览看板

与 brief.html 同屏 · brief-01-dashboard.png · #dashboard

背景周会常问「账上多少、缺口多大」，数却散在 Excel 与网银。

需求一屏对齐头寸 + 趋势 + 结构，能接调拨/融资话茬。

讲解旁白

「这是司库和资金负责人的第一站。请您先看顶部 KPI 与卡片——回答『此刻水位够不够』；再看趋势与结构——钱从哪来、往哪去。下面每一页，都是在这条链上的放大。」

约 30～45 秒

  • 指屏：KPI 带 → 主趋势/结构图 → 若有快捷入口顺带一指。
  • 金句：「先看见脉搏，再谈模型。」
  • 追问：数据从哪来——接集成与同步时间（详细版数据集成屏）。

5.2 资金流

Live · 资金流

brief-02-cashflow.png · #cashflow

背景预测再准，底账对不上就无人信。

需求可追溯流水/单据视图，区分实际与预测生成行。

讲解旁白

「这一屏对准单一事实来源。客户会问预测行和银行流水怎么对——您就指筛选、表体：每一笔能追到来源；预测生成的记录要有标签，不能和实际混成一锅粥。」

约 35～50 秒

  • 指屏：筛选条 → 表格列（类型/是否预测）→ 录入或导入入口。
  • 金句：「底账不信，上层智能全是空话。」

5.3 分析预测

Live · 分析预测

brief-03-analysis.png · #analysis

疑虑「是不是拍脑袋曲线？」要讲清区间、场景、与流水联动。

讲解旁白

「这里回应『黑箱一个数』。请您指时间区间和场景开关——假设摆在台面上；再指主图与下钻——区间多大、数从哪条线汇过来。投委会 / 财委要问的，就是这些东西能不能一眼看见。」

约 40～55 秒

  • 指屏：区间/粒度 → 场景或参数区 → 主图。
  • 合规：主动说失败/无数据时界面提示，不编造余额。

5.4 资金计划

Live · 资金计划

brief-04-plan.png · #plan

背景预测与计划「两张皮」时，业务会说系统没用。

需求版本、审批态、执行对比，闭环到节奏。

讲解旁白

「预测必须能执行。这一屏讲编审与对账：计划不是写完就完——要有版本、要有和实际的偏差。您指列表状态、行项目、对账区，把『预测在 PPT、计划在邮件』的痛点反着讲。」

约 35～50 秒

  • 指屏：版本/状态 → 行项目维度 → 对账或执行反馈。
  • 回扣：接分析屏「曲线落进表格与计划」。

5.5 DataAgent

Live · DataAgent

brief-05-agent.png · ?embed=1&ai=1#dashboard

需求在业务页旁问数，勿跳到孤立聊天窗；回答带出处。

路演ai=1 自动展开；生产由用户手开抽屉。

讲解旁白

「助手在抽屉里，主界面仍是看板——不是另一个产品。路演用 ai=1 省得您多点一下；上线后由用户自己打开。合规底线说一句：建议不等于决裁，查不了就说查不了。」

约 40～50 秒

  • 指屏：主区看板 → 右侧抽屉一轮问答。
  • 技术备注：后端 /api/agent/chat，失败可走降级链（可口述）。
